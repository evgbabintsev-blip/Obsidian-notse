## Шаблоны заметок

### Шаблон MOC-заметки (карта контента)
```markdown
---
created: YYYY-MM-DD
tags: [moc, <тема>]
status: evergreen
type: moc
---

# 🗂 MOC: <Название темы>

## Активные
- [[Заметка 1]] — описание
- [[Заметка 2]] — описание

## Архив
- ...

## Входящие (для обработки)
- ...
```

## Как это работает

### 1. MOC вместо папок (ключевой принцип)
- Каждая папка первого уровня содержит **MOC-заметку** (Map of Content)
- MOC-заметка = оглавление со ссылками на все связанные заметки
- Глубина папок: максимум 2 уровня (папка → заметки)
- Связи между темами — через ссылки, не через вложенность

### 2. Входящие → Обработка → База (поток)
```
Telegram / Браузер / Мысль
        ↓
   📥 Входящие (inbox)
        ↓
   Обработка (Hermes или вручную)
        ↓
   Заметка с метаданными + ссылками
        ↓
   В нужную папку + ссылка в MOC
```

### 3. LLM Wiki (Hermes как компилятор)
```
Тема → Hermes ищет источники → Компилирует в заметку → Связывает с MOC
```

### 4. Журнал с автоагрегацией
```
Ежедневные → cron каждое утро → создаёт шаблон дня
Еженедельные → cron каждое вс → Hermes собирает итоги недели
Ежемесячные → cron 1-го числа → Hermes собирает итоги месяца
```

## Что уже есть vs что добавить

| Компонент | Статус | Действие |
|-----------|--------|----------|
| 📁База знаний | ✅ Есть | Добавить MOC-заметки |
| 📁Система | ✅ Создана сегодня | Заполняется |
| 📅 Дневник | ✅ Есть (3 уровня) | Добавить шаблоны |
| 📦 Шаблоны | ❌ Нет | **Создать** |
| MOC-заметки | ❌ Нет | **Создать для каждой папки** |
| CRM (Контакты) | ❌ Нет | **Создать** |
| LLM Wiki Pipeline | ❌ Нет | **Настроить cron** |

## Ближайшие шаги

1. Создать шаблоны в 📦 Шаблоны
2. Создать MOC-заметки для существующих папок
3. Настроить cron для автосоздания ежедневных заметок
4. Настроить LLM Wiki пайплайн (Hermes → компиляция → Joplin)
