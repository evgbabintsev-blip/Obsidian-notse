## Главный тренд апреля-июня 2026: LLM Wiki (Карпатый)

Андрей Карпатый (ex-Tesla AI, ex-OpenAI) запустил идею: **LLM не отвечает на вопросы, а ведёт связную вики**. Нейросеть компилирует знания из источников в markdown-заметки с перекрёстными ссылками. 5000+ звёзд на GitHub за месяц.

**Три реализации под Obsidian:**
1. `obsidian-wiki` от Ar9av — набор Markdown-инструкций для AI-агента
2. Прямая интеграция через Copilot
3. Кастомные пайплайны

**Вывод для Joplin:** Hermes уже умеет писать в Joplin через SQLite. Мы можем реализовать LLM Wiki прямо сейчас — Hermes читает источники, компилирует знания, создаёт заметки с перекрёстными ссылками `[Title](:/id)`.

## Структура: от PARA к MOC (Maps of Content)

**Эволюция подходов (из опыта сообщества):**
- 2008-2021: PARA (Projects, Areas, Resources, Archives) — работало в Evernote
- 2021-2023: Zettelkasten — атомарные заметки + связи
- 2023-2025: MOC (Maps of Content) + Dataview — агрегация через умные запросы
- 2025-2026: LLM Wiki — AI сам строит связи

**Уровни зрелости базы знаний:**
1. Быстрые заметки (inbox)
2. Ключевые заметки (обработанные)
3. MOC — карты контента (агрегация по темам)
4. Домашняя страница (dashboard)

## Что применить в Joplin

### 1. Плоская структура + MOC
Joplin не имеет Dataview и плагинов → не нужна глубокая иерархия папок.
- Максимум 2 уровня вложенности
- Вместо папок — **MOC-заметки**: страницы-хабы со ссылками `[Тема](:/id)`
- Каждая MOC-заметка = оглавление предметной области

### 2. Дневник с автоагрегацией (как Second Brain Vault)
- Ежедневные заметки → ссылки на недельные → ссылки на месячные
- Hermes может автоматически собирать итоги недели/месяца через cron

### 3. AI Capture Pipeline
```
Telegram → Hermes → SQLite → Joplin → WebDAV → Телефон
```
Hermes читает статью/видео/пост → форматирует → пишет в Joplin с метаданными.

### 4. YAML-фронтматтер
Обсидиан-сообщество стандартизирует фронтматтер:
```yaml
---
created: 2026-06-19
tags: [торговля, крипта]
source: t.me/channel/123
status: черновик / обработано / evergreen
---
```

### 5. Разделение: источники vs результат
- **Zotero** (или Inbox в Joplin) — сырые материалы
- **База знаний** — только обработанные заметки
- Не захламлять базу сохранёнными статьями без обработки

## Итог: 5 правил для Joplin

1. **MOC вместо папок** — хабы-оглавления с ссылками, не глубже 2 уровней
2. **YAML-фронтматтер** — дата, теги, статус, источник
3. **LLM Wiki паттерн** — Hermes компилирует знания в связные заметки
4. **Дневник с автоагрегацией** — день → неделя → месяц → Hermes собирает
5. **Inbox vs База** — сырое отдельно, обработанное отдельно
